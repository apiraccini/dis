API Style Guide — Acme Corp

STANDARDS FOR ALL REST APIS

URL Structure:
  https://api.acme.internal/v1/{resource}

  - Use plural nouns: /users, /documents
  - Nest sub-resources: /documents/{id}/chunks
  - Kebab-case for multi-word: /incident-reports

HTTP Methods:
  GET    — Retrieve (idempotent)
  POST   — Create (not idempotent)
  PUT    — Full replace (idempotent)
  PATCH  — Partial update (not idempotent)
  DELETE — Remove (idempotent)

Pagination:
  Cursor-based: ?cursor=<opaque>&limit=<int>
  Default limit: 20, max: 200

Error Response Envelope:
  {"error": {"code": "ERROR_CODE", "message": "Human-readable", "details": {}}}

  Standard codes:
    400 — VALIDATION_ERROR
    401 — UNAUTHORIZED
    404 — NOT_FOUND
    409 — CONFLICT
    422 — UNPROCESSABLE_CONTENT
    429 — RATE_LIMITED

Versioning:
  Version in URL path (/v1/).
  Breaking changes = new version.
  Deprecated endpoints kept for 3 months.
